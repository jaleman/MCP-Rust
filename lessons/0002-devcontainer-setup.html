Lesson 2

Dev Container Setup

~20 minutes Hands-on — you create the file KUKA MCP Server in Rust

What you're doing and why

Rather than installing Rust on Windows directly, you'll develop inside a dev container — a Docker container that VS Code opens as your coding environment. Rust, Cargo, and your editor extensions are all pre-installed inside it. Your project files stay on your Windows machine and are mounted in, so nothing is lost when the container stops.

Prerequisite
You need Docker Desktop running on Windows and the Dev Containers extension installed in VS Code (ms-vscode-remote.remote-containers). Check now: open a terminal and run docker --version. If that works, you're ready.

The two things a devcontainer needs

A dev container is defined by a single JSON file: .devcontainer/devcontainer.json. It needs two things: [devcontainer.json reference]

1. An image — what software is pre-installed

Microsoft publishes official dev container images. The one you want is: 

"image": "mcr.microsoft.com/devcontainers/rust:latest"

This image comes with Rust, Cargo, rustfmt, and clippy already installed. You don't touch any of that — it's just there.

2. Customizations — VS Code extensions and settings

You specify which VS Code extensions should be auto-installed when the container opens. For Rust development you want three:

  • rust-lang.rust-analyzer — autocomplete, type hints, inline errors. Essential.
  • tamasfe.even-better-toml — syntax highlighting for Cargo.toml.
  • serayuzgur.crates — shows latest crate versions inline in Cargo.toml.

Your turn: create the file

In VS Code, create the folder .devcontainer at the root of your project, then create devcontainer.json inside it with this content: 

{
  "name": "KUKA MCP Server — Rust",
  "image": "mcr.microsoft.com/devcontainers/rust:latest",
  "customizations": {
    "vscode": {
      "extensions": [
        "rust-lang.rust-analyzer",
        "tamasfe.even-better-toml",
        "serayuzgur.crates"
      ],
      "settings": {
        "editor.formatOnSave": true,
        "[rust]": {
          "editor.defaultFormatter": "rust-lang.rust-analyzer"
        }
      }
    }
  },
  "postCreateCommand": "rustc --version && cargo --version"
}

The postCreateCommand runs once after the container is built — it prints the Rust and Cargo versions so you can confirm everything installed correctly.

Open the project in the container

Once the file is saved, reopen the project inside the container:

  1. Open the Command Palette: Ctrl+Shift+P
  2. Type Reopen in Container and press Enter
  3. VS Code will pull the image and build the container — this takes a few minutes the first time
  4. When it opens, you'll see Dev Container: KUKA MCP Server — Rust in the bottom-left corner
Verify it worked
Open a terminal in VS Code (Ctrl+`) and run: 
rustc --version
cargo --version
You should see version numbers for both. If you do, your Rust development environment is fully set up.

What the container is and isn't

┌─────────────────────────────────────────────────────┐ │ Your Windows machine │ │ │ │ ┌─────────────────────────────────────────────┐ │ │ │ Dev Container (Docker) │ │ │ │ │ │ │ │ Rust + Cargo + rust-analyzer │ │ │ │ │ │ │ │ /workspaces/MCP-Rust ◄── mounted from │ │ │ │ (your project files) Windows disk │ │ │ └─────────────────────────────────────────────┘ │ │ │ │ Docker Desktop (runs the container) │ │ VS Code (connects to it via Remote extension) │ └─────────────────────────────────────────────────────┘

Your files live on Windows — the container just provides the tools. If you delete the container, your code is safe. This also means you can commit .devcontainer/ to git and anyone else can open your project in the same environment.
