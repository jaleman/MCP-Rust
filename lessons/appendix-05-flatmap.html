flatMap — Java vs Rust

Reference — return to this when flat_map() is confusing KUKA MCP Server in Rust

The problem flatMap solves

When you call map() and the transformation produces a collection per element, you end up with a nested collection — a list of lists — when you usually just want a flat list. flatMap() maps and then immediately flattens, giving you one stream of all the results merged together.

map()    → [[0, 21, 45], [102, 310]]  ← list of lists (nested)
flatMap() → [0, 21, 45, 102, 310]        ← single flat list

Example 1 — Splitting sentences into words

The simplest case: you have a list of sentences and want all the individual words in one flat list.

Java:

List<String> sentences = List.of("reflectors must", "minimum height");

// map() — gives you a List of arrays (nested, awkward to work with)
List<String[]> nested = sentences.stream()
    .map(s -> s.split(" "))
    .collect(Collectors.toList());
// Result: [["reflectors", "must"], ["minimum", "height"]]

// flatMap() — merges all words into one flat List
List<String> words = sentences.stream()
    .flatMap(s -> Arrays.stream(s.split(" ")))
    .collect(Collectors.toList());
// Result: ["reflectors", "must", "minimum", "height"]

Rust:

let sentences = vec!["reflectors must", "minimum height"];

// map() — gives you an iterator of iterators (nested)
// This is awkward and rarely what you want
let nested: Vec<Vec<&str>> = sentences.iter()
    .map(|s| s.split_whitespace().collect())
    .collect();
// Result: [["reflectors", "must"], ["minimum", "height"]]

// flat_map() — merges all words into one flat Vec
let words: Vec<&str> = sentences.iter()
    .flat_map(|s| s.split_whitespace())
    .collect();
// Result: ["reflectors", "must", "minimum", "height"]

Example 2 — Finding all match positions (from this project)

In Lesson 15, flat_map collects every position where any query term appears in the document. Each term maps to multiple positions, and flat_map merges all those position lists into one.

Java:

List<String> terms = List.of("height", "reflector");
String docLower = "reflector height limit 2000mm, minimum height 150mm";

// Each term maps to a list of positions — flatMap merges them all
List<Integer> positions = terms.stream()
    .flatMap(term -> {
        // Find every occurrence of this term in the document
        List<Integer> found = new ArrayList<>();
        int pos = docLower.indexOf(term);
        while (pos != -1) {
            found.add(pos);
            pos = docLower.indexOf(term, pos + 1);
        }
        return found.stream();
    })
    .sorted()
    .distinct()
    .collect(Collectors.toList());
// Result: [0, 10, 41] — positions of "reflector", "height", "height"

Rust:

let terms = vec!["height", "reflector"];
let lower = "reflector height limit 2000mm, minimum height 150mm";

// match_indices() returns all occurrences as (position, matched_text) tuples.
// flat_map merges the per-term iterators into one stream of positions.
let mut positions: Vec<usize> = terms.iter()
    .flat_map(|term| lower.match_indices(*term).map(|(pos, _)| pos))
    .collect();

positions.sort();   // sort so duplicates are adjacent
positions.dedup();  // remove consecutive duplicates

// Result: [0, 10, 41] — same positions, deduped

Side-by-side summary

JavaRust
stream.flatMap(x -> subStream) iter.flat_map(|x| sub_iter)
stream.distinct() vec.sort(); vec.dedup();
stream.sorted() vec.sort() or iter.collect::<BTreeSet>()

The key difference: Java's flatMap takes a function that returns a Stream. Rust's flat_map takes a closure that returns any type that implements IntoIterator — which includes Vec, arrays, ranges, and all the standard iterators like split_whitespace() and match_indices().
