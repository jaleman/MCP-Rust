Lesson 15

Multiple Excerpts

~20 minutes Hands-on — edit, build, test in your dev container KUKA MCP Server in Rust

What you're fixing

Right now search_docs returns one excerpt per document — the window around the first match. If the Reflector Deployment Guide mentions "height" in five different sections, Claude only sees the first one. The other four are invisible.

This lesson changes the tool to return up to three non-overlapping excerpts per document, one for each distinct region where query terms appear.

Before: one excerpt per document (first match only)

After:  • Reflector Deployment Guide
          ...Reflectors must be mounted at a height of 150–2000 mm...

          ...The maximum height above floor level is 2000 mm...

          ...Height tolerances apply when using angled mounting brackets...

New concepts — translated from Java

Concept 1: match_indices() — finding all occurrences at once

Until now you used str::find(), which returns the position of the first match only. To find every occurrence you need match_indices(), which returns an iterator of (usize, &str) tuples — one per occurrence, each containing the byte position and the matched text. 

let text = "height limit 2000mm, minimum height 150mm";

for (pos, matched) in text.match_indices("height") {
    println!("{pos}: {matched}");
}
// 0: height
// 21: height

Java has no direct equivalent — the closest approach is a loop calling indexOf(term, fromIndex) repeatedly and advancing the position each time. match_indices() packages that loop into a lazy iterator.

Concept 2: flat_map() — one-to-many transformation

.map() transforms each element into one value. When the transformation produces an iterator per element and you want to merge all those iterators into a single flat stream, you use .flat_map(). Java's Stream.flatMap() is identical. See Appendix 5: flatMap for full Java and Rust examples side by side.

JavaRust
terms.stream()
  .flatMap(t -> findAll(doc, t))
  .collect(toList())		 terms.iter()
  .flat_map(|t| lower.match_indices(*t))
  .collect()

In this lesson, each query term maps to an iterator of (position, matched) tuples. flat_map merges all those iterators so you end up with one stream of every match position across all terms.

Concept 3: dedup() — removing consecutive duplicates

Two different query terms might match at the same byte position (e.g. "height" and "heights" both start at position 42). After collecting all positions, you sort them and then call .dedup() to remove consecutive duplicates — leaving each position appearing at most once. 

let mut positions = vec![0, 21, 21, 45]; // 21 appears twice
positions.dedup();
// positions is now [0, 21, 45]
dedup() requires the Vec to be sorted first

dedup() only removes consecutive duplicates. If the Vec is unsorted, duplicates that are not adjacent will survive. Always call .sort() before .dedup() when your goal is removing all duplicates — the same pattern as Java's new ArrayList<>(new LinkedHashSet<>(list)), but more explicit about what is happening.

Step 1 — Replace the excerpt block in search_docs

Open mcp-server/src/main.rs. Inside the if terms.iter().all(...) block, replace everything from the excerpt comment down to the results.push() call:

Before:

// Anchor the excerpt at the first term that has an exact substring match.
// Falls back to the document start if all matches were fuzzy-only.
let pos = terms.iter().find_map(|term| lower.find(*term)).unwrap_or(0);
// 200 chars of lookback context, 400 chars of lookahead
let start = pos.saturating_sub(200);
let end = (pos + 400).min(content.len());
// Slice from the original (not lowercased) content to preserve capitalisation
let excerpt = content[start..end].trim();

// Score = total exact occurrences of all terms; used for ranking
let score: usize =
    terms.iter().map(|term| lower.matches(*term).count()).sum();

results.push((
    score,
    format!("• {title}\n  Source: {resource}\n\n  ...{excerpt}..."),
));

After:

// Collect byte positions of every exact match across all query terms,
// then sort and deduplicate so each position appears at most once
let mut positions: Vec<usize> = terms.iter()
    .flat_map(|term| lower.match_indices(*term).map(|(pos, _)| pos))
    .collect();
positions.sort();
positions.dedup();

// Build up to 3 non-overlapping excerpt windows (150 chars back, 300 ahead)
let mut excerpts: Vec<&str> = Vec::new();
let mut last_end: usize = 0;
for pos in &positions {
    let pos = *pos;
    // Skip positions that fall inside the previous window
    if pos < last_end {
        continue;
    }
    let start = pos.saturating_sub(150);
    let end = (pos + 300).min(content.len());
    excerpts.push(content[start..end].trim());
    last_end = end;
    if excerpts.len() >= 3 {
        break;
    }
}

// Fall back to the document start if all matches were fuzzy-only
if excerpts.is_empty() {
    excerpts.push(content[..400_usize.min(content.len())].trim());
}

// Score = total exact occurrences of all terms; used for ranking
let score: usize =
    terms.iter().map(|term| lower.matches(*term).count()).sum();

results.push((
    score,
    format!("• {title}\n  Source: {resource}\n\n  ...{}...",
        excerpts.join("\n\n  ...")),
));

What changed and why

  • flat_map(|term| lower.match_indices(*term).map(|(pos, _)| pos)) — for each query term, get all its match positions in the document. .map(|(pos, _)| pos) discards the matched text (we only need the position), and flat_map merges the per-term iterators into one flat stream of positions.
  • positions.sort(); positions.dedup(); — sort so duplicates are adjacent, then remove them. Without this, two terms matching at the same location would produce two identical excerpt windows.
  • if pos < last_end { continue; } — the non-overlap guard. If this position falls inside the window we just emitted, skip it. This prevents two excerpts from showing the same passage.
  • 400_usize.min(content.len()) — the underscore in 400_usize is a type suffix, telling Rust this literal is a usize. Without it, Rust would need to infer the type from context, which fails here because the literal appears on the left of .min() with no other type information available.
  • excerpts.join("\n\n ...") — joins multiple excerpt strings with a visual separator so Claude can read them as distinct passages.
