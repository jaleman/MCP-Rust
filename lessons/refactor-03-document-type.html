Refactor Appendix 3

The Document Type — from Raw Strings to a Domain Entity

Refactor step 2 — commit d0b9f07 KUKA MCP Server in Rust

The missing type

After step 1 the code was better organised, but three functions still did the same dance independently: list_docs_in, search_docs_in, and list_resources each walked the directory, checked the extension, read the file into a String, and called extract_frontmatter_field for whichever fields they happened to need — re-scanning the frontmatter block on every call. Repeated code like that is usually a symptom of a missing type: there was no value in the program that meant "a loaded document".

Rust — bundle.rs, new in this step:

/// One loaded markdown document with its frontmatter parsed into fields.
#[derive(Debug)]
pub struct Document {
    pub path: PathBuf,
    pub stem: String,            // filename without .md — doubles as the resource URI id
    pub title: String,
    pub doc_type: String,
    pub resource: String,
    pub description: Option<String>,   // not every doc has one
    pub content: String,         // full file text, frontmatter included
    pub body_start: usize,       // byte offset just past the closing ---
}

impl Document {
    pub fn load(path: &Path) -> Result<Self> { … }   // parse once, here, only here

    /// The document text after the frontmatter block.
    pub fn body(&self) -> &str {
        &self.content[self.body_start..]
    }
}

pub fn load_bundle(dir: &Path) -> Result<Vec<Document>> { … }

The Java equivalent — an entity plus a repository:

public record Document(
        Path path,
        String stem,
        String title,
        String docType,
        String resource,
        Optional<String> description,
        String content,
        int bodyStart) {

    public static Document load(Path path) throws IOException { … }

    public String body() {
        return content.substring(bodyStart);
    }
}

public final class Bundle {
    /** The repository method every consumer calls. */
    public static List<Document> loadBundle(Path dir) throws IOException { … }
}

This is the repository pattern in miniature: consumers ask for domain objects and never touch files or parsing. The three call sites collapsed accordingly — list_docs_in went from a 25-line walk-and-parse loop to "load_bundle, group by doc.doc_type, format"; search_docs_in lost all its frontmatter code; and list_resources in main.rs became a single iterator chain: 

let resources: Vec<Resource> = docs
    .into_iter()
    .map(|doc| {
        let uri = format!("kuka://docs/{}", doc.stem);
        let mut raw = RawResource::new(uri, doc.stem)
            .with_title(doc.title)
            .with_mime_type("text/markdown".to_string());
        if let Some(desc) = doc.description {   // like ifPresent() on Optional
            raw = raw.with_description(desc);
        }
        Annotated::new(raw, None)
    })
    .collect();

Two Rust details worth pausing on

1. body() returns &str, not String. In Java, content.substring(bodyStart) copies the characters into a new String. Rust's &self.content[self.body_start..] copies nothing — it returns a slice: a pointer into the original text plus a length. The borrow checker guarantees the Document outlives the slice, which is why this is safe without a garbage collector (see Appendix 1). With gigabyte documents on the roadmap, "no copies unless asked" is a habit worth building.

2. description is Option<String>. The other fields have sensible fallbacks (title falls back to the filename, type to "uncategorised"), so they're plain Strings. But a missing description is genuinely absent — the resource listing skips it entirely. Encoding that as Option instead of an empty string means the compiler forces every user of the field to handle absence, exactly like Optional done consistently in Java. The type of each field now documents which fallback policy applies.

Compile-time enforcement we got for free

One bug fixed in this step wasn't found by a test — it was found by the compiler. A new test called result.unwrap_err(), which requires the success value (Vec<Document>) to be printable for debugging. The build failed with error[E0277]: `Document` doesn't implement `std::fmt::Debug` until #[derive(Debug)] was added. In Java, every object has a toString() whether useful or not; in Rust, printability is opt-in and checked at compile time — the same derive-what-you-need philosophy as #[derive(Deserialize)] from Lesson 5.
