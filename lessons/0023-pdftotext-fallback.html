Lesson 23

External Commands & the pdftotext Fallback

~25 min Topic: std::process::Command Prerequisite: Lesson 22 (clap)

Not all PDFs are equal. Some have a text layer that pdf-extract — a Rust library — fails to parse correctly, returning an empty string even though the document clearly contains text. This happens with certain font encodings, compression schemes, or non-standard PDF structure that the library doesn't handle.

The fix is a fallback to pdftotext, a command-line tool from the poppler-utils package. Poppler has been maintained for over 20 years and handles far more PDF edge cases than pdf-extract. When pdf-extract comes up empty, trying pdftotext on the same file often succeeds. In this lesson you'll call it from Rust using std::process::Command — Rust's ProcessBuilder equivalent.

What pdftotext cannot fix

If a PDF was created by rasterising slides or pages into images — common when exporting some PowerPoint files — there is no text layer at all. Neither pdf-extract nor pdftotext can help; both return empty output. Extracting text from image-based PDFs requires OCR (e.g. Tesseract), which reads the pixels rather than the PDF structure. That is out of scope for this lesson.

As a bonus, you'll add a --force-pdftotext flag — the payoff that was teased at the end of Lesson 22. With clap already in place, it's literally two lines.

Prerequisite check

This lesson assumes Lesson 22 is applied: clap is in Cargo.toml and extract.rs uses an Args struct with Args::parse(). If you haven't done that yet, do it now before continuing — the steps in this lesson build directly on top of it.

Quick check: cargo build && ./target/debug/extract --help should print a usage page. If it still crashes or says "Usage: extract <pdf-file> …", go back to Lesson 22 first.

Step 0: Install pdftotext

pdftotext ships as part of poppler-utils. In the dev container (or any Debian/Ubuntu machine): 

sudo apt-get install -y poppler-utils

Confirm it's available (pdftotext doesn't support --version — test on a real file):

pdftotext kuka-docs/EmergencyFireAlarm.pdf -
# Expected: text output or blank line — no "command not found"

The tool accepts a PDF path and an output path. Passing - as the output path is the key trick — it writes the extracted text to stdout instead of a file, which is exactly what we want when capturing output from Rust. 

# Manual test — try this on a known-empty PDF before touching Rust code
pdftotext kuka-docs/EmergencyFireAlarm.pdf -

std::process::Command — Rust's ProcessBuilder

In Java you spawn an external process with ProcessBuilder. In Rust the equivalent is std::process::Command. The API is a builder chain — no surprises if you've used picocli or fluent-style Java APIs before.

Java (ProcessBuilder)Rust (std::process::Command)
new ProcessBuilder("pdftotext", path, "-") Command::new("pdftotext").arg(path).arg("-")
pb.start(); stream → bytes .output()?
blocks + captures stdout, stderr, exit status
process.exitValue() == 0 output.status.success()
new String(bytes, "UTF-8") String::from_utf8(output.stdout)?

.output() is the key method. It blocks until the process exits and returns a Result<Output>. The Output struct has three fields:

  • status — the exit code, as an ExitStatus
  • stdout — everything the process wrote to stdout, as Vec<u8>
  • stderr — everything written to stderr, as Vec<u8>

Both stdout and stderr are raw bytes (Vec<u8>) because a process can write arbitrary binary data. Converting to String is an explicit step — String::from_utf8(output.stdout) — because it can fail if the bytes aren't valid UTF-8.

New concept: bail!

You've used ? to propagate errors up the call stack. Sometimes you want to create an error from scratch and return immediately, without a value to unwrap. That's what bail! does — it's a macro from anyhow that returns an Err immediately: 

// Java equivalent: throw new RuntimeException("pdftotext failed: " + stderr);
bail!("pdftotext failed: {stderr}");

Think of it as "throw an anyhow error". The ! marks it as a macro (like println! and format!). It works in any function that returns Result.

Step 1: Add the --force-pdftotext flag

This is the payoff from Lesson 22. To add a new boolean flag to a clap struct, you add one field with a #[arg(long)] attribute: 

#[derive(Parser)]
#[command(name = "extract", about = "Extract KUKA PDFs to OKF markdown", version)]
struct Args {
    /// A single PDF file, or a directory of PDFs
    input: String,

    /// The knowledge/ directory to write OKF files into
    output_dir: String,

    /// Skip pdf-extract and go straight to pdftotext
    #[arg(long)]
    force_pdftotext: bool,   // ← new: two lines, full CLI flag for free
}

With bool, clap treats this as a flag that defaults to false when absent and becomes true when the user passes --force-pdftotext. No argument value needed — just the presence of the flag flips the boolean.

You also need to thread the flag down to process_pdf. Update the calls in main to pass args.force_pdftotext, and add it as a parameter: 

// In main — single-file path:
if let Err(e) = process_pdf(input, knowledge_dir, args.force_pdftotext) { … }

// In main — batch loop:
match process_pdf(&path, knowledge_dir, args.force_pdftotext) { … }

// Function signature:
fn process_pdf(pdf_path: &Path, knowledge_dir: &Path, force_pdftotext: bool) -> Result<()> { … }

Step 2: Write the try_pdftotext function

Add this function to extract.rs. Place it near the top, before process_pdf: 

/// Calls `pdftotext <pdf_path> -` and returns the captured stdout as a String.
/// The `-` argument tells pdftotext to write to stdout instead of a file.
fn try_pdftotext(pdf_path: &Path) -> Result<String> {
    // Build and run the command — .output() blocks until the process exits
    let output = std::process::Command::new("pdftotext")
        .arg(pdf_path)  // the PDF to process
        .arg("-")       // "-" means: write output to stdout
        .output()
        // .context() attaches a human-readable message if the command isn't found
        .context("pdftotext not found — install poppler-utils")?;

    // A non-zero exit code means pdftotext rejected the file
    if !output.status.success() {
        let stderr = String::from_utf8_lossy(&output.stderr);
        bail!("pdftotext failed: {stderr}");
    }

    // stdout is Vec<u8> — convert to String, propagating an error if not valid UTF-8
    String::from_utf8(output.stdout)
        .context("pdftotext output is not valid UTF-8")
}

Note the .context() call after .output(): if pdftotext is not installed, the OS returns an error like "No such file or directory". Without .context() that error would be confusing. With it, the user sees "pdftotext not found — install poppler-utils" instead.

Step 3: Add the fallback in process_pdf

Replace the single pdf_extract::extract_text() call with an if/else that checks the flag and whether the text came back empty: 

fn process_pdf(pdf_path: &Path, knowledge_dir: &Path, force_pdftotext: bool) -> Result<()> {
    println!("Extracting: {}", pdf_path.display());

    // Try pdf-extract first — unless the caller forced pdftotext
    let text = if force_pdftotext {
        // Flag set: skip pdf-extract entirely
        try_pdftotext(pdf_path)?
    } else {
        let extracted = pdf_extract::extract_text(pdf_path)?;
        if extracted.trim().is_empty() {
            // pdf-extract returned nothing — fall back to pdftotext
            println!("  pdf-extract returned empty text — trying pdftotext…");
            try_pdftotext(pdf_path)?
        } else {
            extracted
        }
    };

    // Everything below is unchanged from Lesson 22
    let stem = pdf_path.file_stem().unwrap().to_string_lossy();
    // … rest of the function …
}
Pattern to remember

The if expr { … } else { … } block is an expression in Rust — it has a value. Assigning the whole block to let text = if … { … } else { … }; is idiomatic. In Java you'd need a mutable variable and separate assignment. In Rust the value flows out from whichever branch ran.

Add .context() to Cargo.toml imports

.context() is from the anyhow trait. You already have anyhow = "1" in Cargo.toml, but you need to import the trait and the bail! macro. Change the import line at the top of extract.rs: 

// Before:
use anyhow::Result;

// After — add Context (for .context()) and bail! (for early-exit errors):
use anyhow::{bail, Context, Result};

The complete updated extract.rs

Here is the complete file after all changes. The only things that changed from the Lesson 22 version are: the new import, the new force_pdftotext field, the new try_pdftotext function, and the updated process_pdf signature and body. 

use anyhow::{bail, Context, Result};
use clap::Parser;
use std::path::Path;

/// Extracts KUKA PDFs to OKF markdown files in a knowledge bundle.
#[derive(Parser)]
#[command(name = "extract", about = "Extract KUKA PDFs to OKF markdown", version)]
struct Args {
    /// A single PDF file, or a directory of PDFs
    input: String,

    /// The knowledge/ directory to write OKF files into
    output_dir: String,

    /// Skip pdf-extract and go straight to pdftotext
    #[arg(long)]
    force_pdftotext: bool,
}

fn main() {
    let args = Args::parse();

    let input = Path::new(&args.input);
    let knowledge_dir = Path::new(&args.output_dir);

    if !knowledge_dir.is_dir() {
        eprintln!("Output directory does not exist: {}", knowledge_dir.display());
        std::process::exit(1);
    }

    if input.is_file() {
        if let Err(e) = process_pdf(input, knowledge_dir, args.force_pdftotext) {
            eprintln!("Error: {e}");
            std::process::exit(1);
        }
    } else if input.is_dir() {
        let entries = std::fs::read_dir(input).unwrap_or_else(|e| {
            eprintln!("Cannot read directory: {e}");
            std::process::exit(1);
        });

        let mut ok: usize = 0;
        let mut failed: usize = 0;

        for entry in entries.flatten() {
            let path = entry.path();
            if path.extension().and_then(|e| e.to_str()) != Some("pdf") {
                continue;
            }
            match process_pdf(&path, knowledge_dir, args.force_pdftotext) {
                Ok(()) => ok += 1,
                Err(e) => {
                    eprintln!("Skipping {}: {e}", path.display());
                    failed += 1;
                }
            }
        }

        println!("\nDone: {ok} extracted, {failed} failed.");
    } else {
        eprintln!("Input path not found: {}", input.display());
        std::process::exit(1);
    }
}

/// Calls `pdftotext <pdf_path> -` and returns the captured stdout as a String.
fn try_pdftotext(pdf_path: &Path) -> Result<String> {
    let output = std::process::Command::new("pdftotext")
        .arg(pdf_path)
        .arg("-")
        .output()
        .context("pdftotext not found — install poppler-utils")?;

    if !output.status.success() {
        let stderr = String::from_utf8_lossy(&output.stderr);
        bail!("pdftotext failed: {stderr}");
    }

    String::from_utf8(output.stdout)
        .context("pdftotext output is not valid UTF-8")
}

fn process_pdf(pdf_path: &Path, knowledge_dir: &Path, force_pdftotext: bool) -> Result<()> {
    println!("Extracting: {}", pdf_path.display());

    let text = if force_pdftotext {
        try_pdftotext(pdf_path)?
    } else {
        let extracted = pdf_extract::extract_text(pdf_path)?;
        if extracted.trim().is_empty() {
            println!("  pdf-extract returned empty text — trying pdftotext…");
            try_pdftotext(pdf_path)?
        } else {
            extracted
        }
    };

    let stem = pdf_path.file_stem().unwrap().to_string_lossy();
    let filename = pdf_path.file_name().unwrap().to_string_lossy();
    let slug = stem
        .to_lowercase()
        .replace(' ', "-")
        .replace(['.', '(', ')'], "");

    let okf = format!(
"---
type: technical-note
title: {stem}
description: Extracted from KUKA documentation.
resource: kuka-docs/{filename}
tags: [extracted, technical-note]
timestamp: 2026-06-26T00:00:00Z
---

{body}",
        stem = stem,
        filename = filename,
        body = text.trim(),
    );

    let output = knowledge_dir.join(format!("{slug}.md"));
    std::fs::write(&output, &okf)?;
    println!("  → {}", output.display());

    Ok(())
}
