Lesson 19

Drop and TempDir

~20 minutes Hands-on — edit, build, test in your dev container KUKA MCP Server in Rust

What you're fixing

The integration tests from Lesson 18 call teardown() at the end of each test to delete the temp directory. If a test panics, teardown() is skipped — there is no finally block in Rust — and the directory is left on disk.

This lesson replaces the manual setup/teardown pair with tempfile::TempDir, a type that deletes its directory automatically when it goes out of scope — even during a panic. The mechanism is Rust's Drop trait.

New concept: the Drop trait

Every Rust value is destroyed when it goes out of scope. If the type implements the Drop trait, Rust calls its drop() method first — giving the type a chance to release resources before the memory is freed.

JavaRust
interface AutoCloseable {
  void close();
}		 trait Drop {
  fn drop(&mut self);
}
try (var x = new Resource()) {
  // close() called on exit
}		 { let x = Resource::new();
  // drop() called at end of block
}

The key difference: Java's try-with-resources is opt-in syntax — you must wrap the resource in a try block to get automatic cleanup. Rust's Drop is automatic for every value, every time, with no special syntax required. You cannot accidentally forget it.

Java: close() called only inside try-with-resources
         — forget the try block and close() is never called

Rust: drop() called whenever the value goes out of scope
         — there is no way to forget it

Crucially, drop() is also called during panic unwinding — when a test fails, Rust walks the stack and drops every value in scope before unwinding. This is what makes TempDir reliable: the directory is deleted whether the test passes or panics.

TempDir — Drop in practice

The tempfile::TempDir type creates a uniquely-named temp directory on construction and implements Drop to delete it. You never call teardown manually — you just let the variable go out of scope. 

// TempDir creates the directory when constructed
let temp_dir = tempfile::TempDir::new().unwrap();

// .path() returns a &Path to the directory
println!("{}", temp_dir.path().display());

// At the end of this block, Drop is called and the directory is deleted —
// even if this block was exited via a panic

Step 1 — Add the tempfile crate

Run this in your dev container from the mcp-server directory: 

cargo add tempfile

This adds tempfile to Cargo.toml under [dependencies]. Since it is only used in tests you could add it under [dev-dependencies] instead — dev dependencies are excluded from the production binary. For now [dependencies] is fine.

Step 2 — Update the integration test module

Open mcp-server/src/main.rs. Replace the entire mod integration_tests block with this updated version: 

#[cfg(test)]
mod integration_tests {
    use super::*;
    use std::fs;

    // Creates a uniquely-named temp directory and writes one known OKF document into it.
    // Returns TempDir — not PathBuf — so the caller owns the directory and controls
    // when it is dropped. When TempDir goes out of scope, Drop deletes it automatically.
    fn setup_test_bundle() -> tempfile::TempDir {
        // TempDir::new() creates a unique directory in the system temp folder
        let temp_dir = tempfile::TempDir::new().unwrap();

        // A minimal but valid OKF document containing known searchable text
        let doc = "\
---
type: technical-note
title: Reflector Guide
description: Test document for integration tests.
resource: kuka-docs/test.pdf
tags: [test]
timestamp: 2026-01-01T00:00:00Z
---

Reflectors must be mounted at a height of 150 to 2000 mm above floor level.
The maximum spacing between reflectors is 8 metres.";

        fs::write(temp_dir.path().join("reflector-guide.md"), doc).unwrap();

        // Return TempDir — Drop is NOT called here because ownership moves to the caller
        temp_dir
    }

    #[test]
    fn search_finds_matching_document() {
        // temp_dir owns the directory; it is deleted when this variable goes out of scope
        let temp_dir = setup_test_bundle();
        unsafe { std::env::set_var("KUKA_KNOWLEDGE_DIR", temp_dir.path()); }

        let server = KukaServer::new();
        let input = SearchInput { query: "reflector height".to_string() };
        let result = server.search_docs(Parameters(input)).unwrap();

        let output = format!("{:?}", result);
        assert!(output.contains("Reflector Guide"), "Should find the document by title");
        assert!(output.contains("150"), "Should include excerpt containing the height value");

        // temp_dir goes out of scope here — Drop called, directory deleted automatically
    }

    #[test]
    fn search_returns_no_results_for_unknown_query() {
        let temp_dir = setup_test_bundle();
        unsafe { std::env::set_var("KUKA_KNOWLEDGE_DIR", temp_dir.path()); }

        let server = KukaServer::new();
        let input = SearchInput { query: "hydraulic pump".to_string() };
        let result = server.search_docs(Parameters(input)).unwrap();

        let output = format!("{:?}", result);
        assert!(output.contains("No results found"), "Unrelated query should return no results");
    }

    #[test]
    fn list_docs_shows_document_grouped_by_type() {
        let temp_dir = setup_test_bundle();
        unsafe { std::env::set_var("KUKA_KNOWLEDGE_DIR", temp_dir.path()); }

        let server = KukaServer::new();
        let result = server.list_docs().unwrap();

        let output = format!("{:?}", result);
        assert!(output.contains("Reflector Guide"), "Should list the document title");
        assert!(output.contains("technical-note"), "Should group by type from frontmatter");
    }
}

What changed

  • setup_test_bundle() now returns tempfile::TempDir instead of PathBuf. The directory path is accessed via .path() rather than the variable directly.
  • teardown() and all calls to it are gone. Cleanup is handled by TempDir's Drop implementation.
  • The comment "temp_dir goes out of scope here" marks where drop() is implicitly called. You do not write anything — Rust handles it.

Step 3 — Run the tests

cargo test integration -- --test-threads=1

All three integration tests should pass and no temp directories are left behind, even if a test were to fail.

Quick check

Primary source

std::ops::Drop — Rust standard library docs
Documents the Drop trait. Pay attention to the section on drop order — Rust drops values in the reverse order they were declared, which matters when one value borrows from another. Also note that you can call std::mem::drop(value) to drop a value early, before the end of its scope.
