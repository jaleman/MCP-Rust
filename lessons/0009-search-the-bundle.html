Lesson 9

Search the Bundle

~25 minutes Hands-on — dev container + AI client KUKA MCP Server in Rust

What you're building

The search_docs tool has returned a stub message since Lesson 5. This lesson replaces it with real search: the tool reads every .md file in knowledge/, finds those whose content matches the query, and returns their titles and source citations.

By the end, asking your AI client "what are the reflector deployment requirements?" will return results drawn from the actual KUKA documentation.

If you know Java: the concepts in this lesson translated

Vec<String> — ArrayList<String>

Vec<T> is Rust's growable list — identical in purpose to Java's ArrayList<T>. The syntax for creating one and adding to it:

JavaRust
List<String> results = new ArrayList<>(); let mut results: Vec<String> = Vec::new();
results.add("item"); results.push("item".to_string());
String.join("\n\n", results) results.join("\n\n")

The key difference from Java: in Rust, all variables are immutable by default. You must write let mut to get a variable you can modify. In Java, local variables are mutable by default and you opt into immutability with final. Rust is the opposite — immutable by default, mut to opt in. You cannot call push() on a Vec without mut.

std::fs::read_dir — Files.list() from java.nio.file

JavaRust
Files.list(path) returns Stream<Path> std::fs::read_dir(path) returns Result<ReadDir>
throws IOException if the directory doesn't exist returns Err(...) if the directory doesn't exist

if let Ok(entries) — try-catch as an expression

In Java you wrap fallible operations in try-catch. In Rust, if let Ok(value) = expr is the idiomatic way to handle a Result when you want to skip a block on failure rather than propagate the error:

JavaRust
 
try {
  var entries = Files.list(dir);
  // use entries
} catch (IOException e) {
  // skip — no results
}
 
if let Ok(entries) = read_dir(dir) {
  // use entries
}
// if Err, the block is simply skipped

.flatten() — filtering out errors from an iterator

read_dir returns an iterator where each item is Result<DirEntry> — it can fail per-file if the OS denies access. .flatten() skips Err values and unwraps Ok ones. The Java stream equivalent:

JavaRust
stream.filter(Objects::nonNull) (after catching per-item errors) iterator.flatten() — drops any Err item silently

Option::and_then() — Optional.flatMap()

path.extension() returns Option<&OsStr> — either Some(extension) or None if the path has no extension. To then convert OsStr to &str (which can also fail), you chain .and_then():

JavaRust
Optional.ofNullable(ext).flatMap(e -> Optional.ofNullable(e.toStr())) path.extension().and_then(|e| e.to_str())

and_then on Option is identical to flatMap on Optional: apply the closure if Some, return None if either the original or the closure is None. [Option::and_then — docs]

Closures |e| e.to_str() — lambda expressions

Rust closures use |param| body syntax. Java lambdas use param -> body. They are the same concept — anonymous functions passed as arguments:

Java lambdaRust closure
e -> e.toString() |e| e.to_str()
(a, b) -> a + b |a, b| a + b

New concepts: reading a directory

std::fs::read_dir

std::fs::read_dir(path) returns an iterator over the entries in a directory. Each entry is a Result<DirEntry> — it can fail if the OS denies access to an individual file. You handle this with .flatten(), which silently skips any Err entries and unwraps the Ok ones: 

for entry in std::fs::read_dir("knowledge").flatten() {
    // entry is a DirEntry — errors are skipped
}

Filtering by extension

path.extension() returns Option<&OsStr>. You chain .and_then() to convert it to Option<&str> and then compare: 

if path.extension().and_then(|e| e.to_str()) != Some("md") {
    continue; // skip non-markdown files
}

and_then on an Option applies a closure if the value is Some, returning None if either the original or the closure returns None. [Option::and_then — docs]

Step 1 — Add a frontmatter helper

Add this free function above the struct KukaServer definition in src/main.rs. It extracts a single field from OKF frontmatter using only the standard library — no YAML crate needed. 

fn extract_frontmatter_field(content: &str, field: &str) -> Option<String> {
    let inner = content.strip_prefix("---\n")?.split_once("\n---")?.0;
    for line in inner.lines() {
        if let Some(value) = line.strip_prefix(&format!("{field}: ")) {
            return Some(value.trim().to_string());
        }
    }
    None
}

strip_prefix("---\n")? returns None (aborting the function) if the file doesn't start with a frontmatter fence. split_once("\n---") splits on the closing fence and takes the part before it. The loop then scans each line for the requested field.

Step 2 — Replace the search_docs stub

Replace the entire search_docs method inside the #[tool_router] impl KukaServer block: 

#[tool(description = "Search KUKA robot documentation for a given query")]
fn search_docs(
    &self,
    Parameters(input): Parameters<SearchInput>,
) -> Result<CallToolResult, McpError> {
    let knowledge_dir = std::path::Path::new("knowledge");
    let query = input.query.to_lowercase();
    let mut results: Vec<String> = Vec::new();

    if let Ok(entries) = std::fs::read_dir(knowledge_dir) {
        for entry in entries.flatten() {
            let path = entry.path();
            if path.extension().and_then(|e| e.to_str()) != Some("md") {
                continue;
            }
            if let Ok(content) = std::fs::read_to_string(&path) {
                if content.to_lowercase().contains(&query) {
                    let title = extract_frontmatter_field(&content, "title")
                        .unwrap_or_else(|| {
                            path.file_stem().unwrap().to_string_lossy().to_string()
                        });
                    let resource = extract_frontmatter_field(&content, "resource")
                        .unwrap_or_default();
                    results.push(format!("• {title}\n  Source: {resource}"));
                }
            }
        }
    }

    let text = if results.is_empty() {
        format!("No results found for '{}'.", input.query)
    } else {
        format!(
            "Found {} result(s) for '{}':\n\n{}",
            results.len(),
            input.query,
            results.join("\n\n")
        )
    };

    Ok(CallToolResult::success(vec![Content::text(text)]))
}

Walking through the code

let mut results: Vec<String> = Vec::new() creates an empty, growable list of strings. The mut is required because push modifies it.

if let Ok(entries) = std::fs::read_dir(...) gracefully handles a missing knowledge/ directory — if the path doesn't exist the whole block is skipped and the tool returns "No results found."

results.join("\n\n") concatenates the result strings with a blank line between each one, producing readable output for the AI client.

Working directory note

Path::new("knowledge") is relative to the server's working directory. When Copilot or Claude Code launches the binary, the working directory is the workspace root /workspaces/MCP-Rust/, so knowledge/ resolves correctly. If you get "No results found" on a query that should match, run pwd in the terminal where the server starts to confirm the working directory.
